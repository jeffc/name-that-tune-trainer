# Agent Guidelines & Rules

## Project Context

*   **Workspace**: `name-that-tune`
*   **Stack**: React, TypeScript, Vite, CSS, local storage.

## General Rules

*   Maintain documentation integrity. Do not modify unrelated comments. Document
    code verbosely and thoroughly, so that someone unfamiliar with the codebase
    can easily understand what's going on.
*   Ensure zero server-side requirements for deployment (pure client-side execution).
*   Follow clean coding and modularization patterns.
*   In addition to code comments, be sure to keep the README, AGENTS, and DESIGN
    files up to date as the project evolves.

## Workflow rules

*   Assume that the user will handle repository / commit management unless they
    ask you explicitly to do so. Interpret any such request as restrictively as
    is reasonably possible: the user asking you to make and push a single commit
    is not permission to do so a second time without asking first. The user will
    want to verify that code works before committing it to the repository.
